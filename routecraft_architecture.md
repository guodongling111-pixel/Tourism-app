# RouteCraft（行程织）架构设计

## 1️⃣ 系统整体架构概览

```
┌─────────────┐       ┌─────────────┐
│   前端 Web   │  ←→  │   API 网关   │
│ React/TS +  │       │  FastAPI    │
│ TailwindCSS │       └─────┬───────┘
└─────┬───────┘             │
      │                     │
      │                     │
      ▼                     ▼
┌─────────────┐      ┌─────────────┐
│ 路线生成服务 │      │ 外部数据抓取 │
│ (路线算法)   │      │ 小红书/攻略  │
│ FastAPI + Celery │  │ Selenium + BS4│
└─────────────┘      └─────┬───────┘
                            │
                            ▼
                     ┌─────────────┐
                     │   数据库    │
                     │ PostgreSQL  │
                     │ + PostGIS   │
                     └─────────────┘
```

## 2️⃣ 模块设计

### 前端模块（React + TypeScript + TailwindCSS + 高德/Mapbox）
- 功能：
  - 用户输入目的地、住宿、游玩天数
  - 景点/餐厅选择
  - 小红书链接导入显示候选景点
  - 时间轴 + 地图路线展示导航
  - 行程拖拽调整
- 特点：
  - 组件化开发，易维护
  - TypeScript 提供类型检查，减少 bug
  - Mapbox/高德用于路径和地图展示

### 后端模块（FastAPI + Celery + Redis）

#### 功能
- 提供 RESTful API 服务
- 路线生成服务：
  - 基于最近邻算法优化行程路径
  - 支持插入餐饮节点
  - 支持多日行程的景点自动分配
- 异步任务：攻略内容解析模块

---

#### 当前实现
基于用户粘贴的攻略文本进行内容解析与结构化处理：

- 对用户输入文本进行信息拆解（地点、行程、标签等）
- 构建标准化数据结构，用于后续行程规划与推荐

---

#### 架构设计（规划中）
为扩展数据来源能力，设计了攻略抓取与处理方案：

- 动态网页抓取：Selenium
- 静态内容解析：Requests + BeautifulSoup
- 异步任务调度：Asyncio 提升抓取效率

> 说明：当前阶段优先验证产品核心功能，暂未接入外部平台爬虫能力，  
> 后续可在合规前提下逐步扩展数据获取能力。

---

#### 特点
- 基于 Celery + Redis 的异步任务架构，提升系统在高并发场景下的稳定性
- 模块化设计，支持后续扩展 AI 个性化推荐能力
- 支持生成结构化行程结果（如 Markdown），便于用户直接使用与分享

### 数据库模块（PostgreSQL + PostGIS）
- 功能：
  - 存储景点、餐厅、行程、地图坐标信息
  - 提供地理计算支持：距离计算、最近邻排序
- 特点：
  - 稳定可靠，商用成熟
  - PostGIS 减少自定义地理算法实现



### 部署与运维（Docker + Nginx + Gunicorn + 云服务器）
- 功能：
  - Docker 容器化部署，保证环境一致性
  - Nginx 反向代理 + 静态资源服务
  - Gunicorn 部署 FastAPI 后端
  - 云服务器按需扩容，支持未来多城市扩展
- 特点：
  - 标准商用方案，成熟稳定
  - 易扩展，适合中小规模商业项目

## 3️⃣ 数据流说明

1. 用户操作流程：
   1. 用户在前端输入信息、选择景点/餐厅、导入小红书链接
   2. 前端调用 FastAPI 提供的 REST API
   3. 后端接收请求，调度路线生成算法
   4. 异步任务调用 Celery + Redis 抓取小红书或其他攻略
   5. 路线生成完成后返回前端
   6. 前端展示时间轴 + 地图路线，用户可拖拽调整

2. 数据存储流程：
   - 景点、餐厅、攻略信息存入 PostgreSQL + PostGIS
   - 路线生成结果临时存储或缓存，提高二次访问速度

3. 外部数据抓取流程：
   - Celery 异步任务触发 Selenium/Requests 抓取动态/静态网页
   - 解析后存入数据库
   - 前端展示候选景点，供用户选择加入行程

## 4️⃣ 技术选型原则

1. 技术主流、社区活跃 → 遇到问题容易找到解决方案
2. 工具链稳定 → 商用可靠、坑少
3. 匹配项目规模 → MVP 快速上线，未来易迭代

## 5️⃣ 可扩展点

- AI 个性化推荐：后端集成模型预测用户偏好
- 社交分享：生成行程卡图片，通过前端或云函数渲染
- 多城市支持：数据库与地图服务可扩展

## 6️⃣ 总结

- 架构清晰，前后端分离
- 异步任务支持高并发爬虫与路线计算
- 数据库支持地理计算，减少自定义算法工作量
- 部署方案成熟，商业可用，易扩展
